## Quick Start

### Basic Simulation
```bash
# Run BB84 protocol simulation
python main.py --protocols BB84 --bits 1000 --messages 10

# Run both BB84 and E91 protocols
python main.py --protocols BB84 E91 --bits 2000 --messages 20

# Enable attack simulation
# Edit config/config.yaml to set attacker.eve_present: true
python main.py --attacks 10
```

### Advanced Usage
```bash
# Full simulation with classical comparison
python main.py --protocols BB84 E91 --compare-classical --output-dir my_results

# Custom configuration
python main.py --config my_config.yaml --bits 5000 --messages 50
```

## Configuration

The simulation is configured via `config/config.yaml`:

```yaml
simulation:
  random_seed: 42
  num_runs: 10
  verbose: true

quantum_channel:
  loss_rate_db_per_km: 0.1
  distance_km: 20.0
  quantum_error_rate: 0.05

qkd_protocol:
  key_length_bits: 256
  error_correction: cascade
  privacy_amplification: universal_hash

network_topology:
  nodes:
    - name: Alice
      type: source
    - name: Bob
      type: destination
    - name: Charlie
      type: intermediate
    - name: Repeater
      type: repeater

attacker:
  eve_present: true
  interception_rate: 0.1
  detection_threshold: 0.11
```

## API Usage

### Basic QKD Simulation
```python
from main import QKDNetworkSimulator

# Create simulator
simulator = QKDNetworkSimulator()

# Run BB84 protocol
bb84_results = simulator.run_qkd_protocol("BB84", num_bits=1000)
print(f"Key Rate: {bb84_results['key_rate']:.1f} bits/s")
print(f"QBER: {bb84_results['qber']:.4f}")

# Run secure communication
comm_results = simulator.run_secure_communication(num_messages=10)
print(f"Success Rate: {comm_results['success_rate']:.2f}")
```

## Results and Analysis

### Output Files
- `results/simulation_report.txt`: Comprehensive text report
- `results/simulation_metrics.csv`: Raw metrics data
- `results/plots/`: Generated visualization plots
- `results/simulation_results_*.png`: Performance graphs

### Key Metrics
- **Key Rate**: Bits per second of secure key generation
- **QBER**: Quantum Bit Error Rate (should be < 0.11 for security)
- **Efficiency**: Ratio of final key bits to raw bits
- **Latency**: End-to-end message delivery time
- **Throughput**: Data transmission rate
- **Detection Rate**: Percentage of attacks successfully detected

## Threat Model

### Attacker Capabilities
- **Eve (Eavesdropper)**: Can intercept qubits and perform intercept-resend attacks
- **Network Attacks**: Can manipulate classical communication channels
- **Side-channel Attacks**: Can attempt timing and power analysis

### Security Assumptions
- Quantum channel provides information-theoretic security
- Classical channel is authenticated but not secret
- Network nodes are trusted (no insider attacks)
- Random number generation is truly random

### Attack Detection
- Statistical QBER analysis
- Key rate monitoring
- Anomaly detection algorithms
- Information leakage quantification
